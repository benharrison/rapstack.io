Overview

The RAP stack is an architectural pattern that utilizes PHP & Apache as a thin-client user-interface over more sophisticated back-end REST APIs. An alternative to LAMP, this approach greatly reduces the amount of code in a PHP application, while empowering Web Designers & Software Engineers to focus on their respective strengths.

3-Tier Approach

Presentation Layer

PHP THIN-CLIENT

  • Interacts with REST API for all input/output
  • Contains no business logic or direct database connectivity
Application Layer / Business Logic

REST API

  • Contains all business logic
  • Responsible for database connectivity
Can be built using the tools of your team's preference:
  • C# & WebApi
  • Python & Flask
  • Java & Spring
  • etc
Database Layer

PERSISTENT STORAGE

PHP presentation-layer has no knowledge of the underlying database(s) since all communication goes through the REST APIs.

Can be built using the tools of your team's preference:
  • PostgreSQL
  • MariaDB / MySQL
  • Mongo
  • Cassandra
  • etc
